# Q1-readiness audit

This is an evidence audit, not a prediction of editorial acceptance.

| Criterion | Status | Evidence / action |
|---|---|---|
| Novel, consequential question | PASS | Residual provenance and selective retrieval are isolated. |
| Fresh heterogeneous evidence | PASS | Ten collections and 234 deterministically selected series. |
| Strong references | PASS | Official PatchTST architecture, official Chronos-Bolt, and classical controls. |
| Confirmatory separation | PASS | Hashed selection precedes one sealed confirmation run. |
| Uncertainty and practical effect | PASS | Hierarchical intervals and a fixed 1% SESOI. |
| Negative/leakage controls | PASS | Self-inclusion, provenance, shuffled, random, and oracle diagnostics. |
| Writing depth | PASS | Canonical paper contains 5,606 words. |
| Journal visuals | PASS | Thirty registered vector PDFs. |
| Independent external reproduction | OPEN | Not yet completed; recommended before submission. |
| Target-journal formatting | OPEN | Generic LaTeX is supplied; venue template remains to be chosen. |
| Public DOI and author metadata | OPEN | Release placeholders must be replaced after repository publication. |

## Honest verdict

The package is capable of supporting a Q1 submission because it has a defensible question, strong comparisons, a locked confirmatory design, transparent null-result rules, and a complete reproducibility trail. It is not submission-complete until the three OPEN items are handled, and no artifact can guarantee Q1 acceptance.
