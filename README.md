# Sv1: leakage-resistant retrieval forecasting audit

Sv1 is a clean, preregistered study of whether retrieval-augmented forecasts survive causal memory
construction, forward cross-fitted residuals, explicit abstention, strong references, and a
practically meaningful effect threshold. It was created as a separate repository and does not
overwrite or reinterpret the earlier `S` experiment.

## Registered outcome

The single sealed confirmation run was **inconclusive**. Relative to the three-seed PatchTST
ensemble, the selected cross-fitted retrieval router reduced macro MSE by **1.25%**, with a 95%
hierarchical moving-block bootstrap interval of **-0.31% to 4.08%**. It routed **19.4%** of
confirmation windows and improved **5 of 10** datasets. The point estimate exceeded the registered
1% smallest effect of interest, but the interval included zero and the six-dataset replication rule
was not met; therefore the preregistered positive claim failed. This null-compatible result is the
paper's result, not a target to be optimized after confirmation.

See the [`confirmation decision`](CONFIRMATION_DECISION.json),
[`executive summary`](reports/EXECUTIVE_SUMMARY.md), and
[`Q1-readiness audit`](reports/Q1_READINESS_AUDIT.md) before interpreting the benchmark.

The benchmark uses ten fresh Monash collections (234 selected series), the official Hugging Face
PatchTST architecture, official Amazon Chronos-Bolt-small, AutoTheta, naive references,
RAFT/PIR-aligned transparent adapters, negative controls, three training seeds, hierarchical
moving-block inference, and an exact self-inclusion stress test. Exact upstream commits and
implementation-fidelity limits are registered before confirmation.

## Scientific contract

- All target roles are chronological and disjoint.
- Scaling uses statistics available by the 50% fit boundary.
- Safe residual values are predictions from models trained strictly before their held-out windows.
- Utility fitting, threshold selection, development selection, and confirmation are separate roles.
- The chosen candidate and protected artifacts are content-hashed before confirmation is opened.
- A positive claim requires a 95% interval above zero, an effect of at least 1%, six dataset wins,
  robustness on MAE/MASE, and a valid integrity lock.
- No positive result is assumed or manufactured.

The frozen contract is in [`protocols/DEVELOPMENT_PROTOCOL.md`](protocols/DEVELOPMENT_PROTOCOL.md),
and official-versus-adapter status is explicit in
[`protocols/BASELINE_FIDELITY.csv`](protocols/BASELINE_FIDELITY.csv).

## Reproduce

```bash
python3 -m venv --system-site-packages .venv
.venv/bin/pip install -e ".[dev]"
make check
make download
make prepare
make develop
make lock-confirmation
make confirm
make report
make notebook
make verify
```

`make develop` may train 60 compact PatchTST checkpoints (30 final members and 30 forward-holdout
members) and downloads the 48M-parameter Chronos-Bolt-small checkpoint. Generated source data and
checkpoints are deliberately excluded from Git; manifests, hashes, scalar results, manuscript,
executed notebook, and all figures are versioned.

Run individual stages with `sv1 status`, `sv1 develop`, `sv1 lock-confirmation`, `sv1 confirm`,
`sv1 report`, `sv1 notebook`, and `sv1 verify`. Confirmation refuses to run without a matching lock.

## Evidence package

After completion, the main entry points are:

- `CONFIRMATION_DECISION.json` — machine-readable preregistered decision;
- `reports/EXECUTIVE_SUMMARY.md` — concise result and interpretation boundary;
- `manuscript/paper.md` and `manuscript/paper.tex` — canonical full paper sources;
- `manuscript/supplementary.md` — all 30 registered figures and purposes;
- `results/tables/` — dataset, window, horizon, calibration, bootstrap, and audit outputs;
- `notebooks/Sv1_final_evidence.ipynb` — executed read-only evidence walkthrough;
- `reports/VERIFICATION_REPORT.md` and `ARTIFACT_MANIFEST.json` — release audit.

## Scope and claim limits

Sv1 adapters are not exact reproductions of RAFT, PIR, or TS-RAG. Their names indicate conceptual
alignment only; the fidelity matrix gives the allowed description. Q1 journal readiness is audited
in `reports/Q1_READINESS_AUDIT.md` after the run. The audit can support a submission decision but
cannot guarantee editorial acceptance. Replace anonymous authors, repository owner, and DOI
placeholders only after choosing the actual public release and journal.
